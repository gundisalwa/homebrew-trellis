# kodhama/tap

The **kodhama family's** Homebrew tap — one tap, N formulas. Each product in the
family that ships a real binary owns and pushes its own formula here; this repo
doesn't build anything itself. (Why the tap is shared across the family rather
than per-product: `kodhama-0001-family-delivery` in
[kodhama/kodhama](https://github.com/kodhama/kodhama).)

## trellis — deprecated

The `trellis` formula is **deprecated and frozen at v0.2.29**: the trellis
end-user binary channel retired per **kodhama-0007 rule 5** ("one render, many
copiers"; implemented in
[kodhama/trellis#120](https://github.com/kodhama/trellis/issues/120)).
`brew install kodhama/tap/trellis` still works but warns, and will not receive
updates. Install Trellis instead via:

- **Claude Code** (primary): `/plugin marketplace add kodhama/kodhama` →
  `/plugin install trellis@kodhama` → `/trellis:setup`
- **Any other harness**: the manual copy path — copy the pre-rendered payload;
  see "Get started" in [kodhama/trellis](https://github.com/kodhama/trellis).

Its regeneration machinery (`update-formula.yml` + `scripts/gen-formula.sh`,
trellis decision-0032/0041) is removed; the formula is hand-frozen.

## How future formulas stay current

A formula is a derived resource, regenerated by its own product's release
pipeline — never hand-edited here. The pattern (a `repository_dispatch` from the
product's release workflow re-pins version + sha256): see `decision-0032` in the
[Trellis repo](https://github.com/kodhama/trellis) — retired for trellis itself,
available to any future kodhama product that ships a binary.
